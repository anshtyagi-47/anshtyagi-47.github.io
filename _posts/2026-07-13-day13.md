Today I explored **React Hooks**, one of the most powerful features introduced in React for managing state and other functionalities within functional components. Hooks make it possible to use React features without writing class components, resulting in cleaner and more readable code.

I began by understanding what hooks are and why they were introduced. Before hooks, many advanced React features were only available in class components. Hooks allow developers to manage state, handle side effects, and share reusable logic while keeping components simple and functional.

During the session, I reviewed commonly used hooks such as **`useState`** and received an introduction to **`useEffect`**, learning when each hook is appropriate. I also understood the basic rules of hooks, such as calling them only at the top level of a functional component and never inside loops, conditions, or nested functions.

To reinforce these concepts, I practiced using hooks in small React examples and observed how they make components easier to manage and maintain. This gave me a better understanding of how hooks form the foundation of modern React development.

### What I Learned

* Introduction to React Hooks
* Why hooks were introduced
* Difference between class components and functional components
* Revisiting the `useState` hook
* Basic understanding of the `useEffect` hook
* Rules for using hooks
* Writing cleaner and more maintainable React components

Today's session helped me understand why React Hooks have become the standard approach for modern React development. They simplify component logic, improve code readability, and make it easier to build scalable applications. This foundation will help me explore individual hooks in greater detail in the upcoming sessions.
