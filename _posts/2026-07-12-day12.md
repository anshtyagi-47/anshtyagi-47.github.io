Today I learned different ways to style React applications. While React is mainly responsible for building the user interface, styling is handled using CSS, and I explored a few common approaches to apply styles to components.

I learned how to import CSS files into components, use the `className` attribute instead of `class`, and organize styles to keep the project clean and maintainable. I also looked at inline styling in React and understood when it can be useful compared to traditional CSS.

To practice, I styled a few components by changing layouts, colors, spacing, and typography. This helped me understand how React and CSS work together to create responsive and visually appealing user interfaces.

### What I Learned

- Importing CSS into React components
- Using `className` instead of `class`
- Applying inline styles
- Organizing component styles
- Styling reusable components
- Improving the appearance of React applications

Today's session reminded me that writing clean CSS is just as important as writing clean React code. With a better understanding of styling, I can now build interfaces that are not only functional but also visually appealing.
