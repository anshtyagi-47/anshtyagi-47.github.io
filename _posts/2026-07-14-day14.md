Today I explored the **`useEffect`** hook, one of the most important hooks in React for handling side effects within functional components. Unlike `useState`, which is used to manage component state, `useEffect` allows components to perform tasks such as fetching data, updating the document title, or responding to changes in state and props.

I began by understanding the purpose of the `useEffect` hook and its syntax. I learned that it accepts a callback function and an optional dependency array, which controls when the effect is executed. By experimenting with different dependency arrays, I observed how the hook behaves when it runs after every render, only once when the component mounts, or whenever specific values change.

During the practical session, I used `useEffect` to fetch data from a public API and display the results in a React component. This helped me understand how asynchronous operations can be handled within React while keeping the user interface updated automatically when new data is received.

I also learned the importance of cleaning up side effects, such as removing event listeners or clearing timers, to prevent memory leaks and improve application performance.

### What I Learned

* Introduction to the `useEffect` hook
* Understanding side effects in React
* Using the dependency array effectively
* Fetching data from an API
* Handling asynchronous operations
* Cleaning up side effects
* Building more dynamic React applications

Today's session gave me a solid understanding of how the `useEffect` hook works and why it is essential in modern React development. Learning to fetch data and manage side effects has prepared me to build more interactive and data-driven applications in the upcoming sessions.
