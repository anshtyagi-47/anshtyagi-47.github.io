Today I explored some of the more powerful features of TypeScript that make code easier to organize and reuse. The main focus was on **interfaces** and **type aliases**, which help define the shape of objects and avoid repeating the same type definitions throughout a project.

I learned how interfaces can be used to describe the structure of objects and how they make the code more readable, especially when working with functions or larger applications. I also looked at type aliases and understood when they can be used as an alternative to interfaces for defining custom types.

The lesson also introduced concepts like **union types**, **literal types**, and **type assertions**. These features provide more flexibility while still maintaining TypeScript's type safety. I practiced creating a few examples to understand how these concepts work together.

### What I Learned

- Creating and using interfaces
- Type aliases and their use cases
- Union and literal types
- Type assertions
- Reusing custom types in multiple places
- Writing more structured and maintainable TypeScript code

Today's session helped me understand how TypeScript scales well for larger projects by encouraging better code organization. With the basics now covered, I feel more confident moving toward building real applications with TypeScript.
