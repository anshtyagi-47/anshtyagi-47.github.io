Today I continued my React learning journey by understanding the core building blocks of every React application—**components** and **JSX**.

I learned that React applications are built using reusable components, which makes the code more organized and easier to maintain. Instead of writing a complete page in one file, we can divide it into smaller components that each handle a specific part of the UI.

I also explored JSX, which looks similar to HTML but is actually JavaScript syntax used to describe the user interface. I learned how to embed JavaScript expressions inside JSX using curly braces and understood some of the JSX rules, such as returning a single parent element and using `className` instead of `class`.

Along with the theory, I practiced creating functional components and rendering them inside the application. Seeing how multiple components work together made React's approach much clearer.

### What I Learned

- What React components are
- Functional components
- Introduction to JSX
- Embedding JavaScript expressions in JSX
- JSX rules and syntax
- Rendering components inside other components

Today's lesson helped me understand how React structures applications using reusable pieces of code. The concept of components feels simple yet powerful, and I'm excited to continue learning about props, state, and building more interactive applications.
