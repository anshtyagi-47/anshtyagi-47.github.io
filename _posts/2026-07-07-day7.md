Today I learned about **props (properties)**, one of the core concepts in React that enables communication between components. Props allow data to be passed from a parent component to a child component, making components more dynamic and reusable.

I started by understanding how props work and why they are important in component-based development. Instead of hardcoding values inside a component, props allow the same component to display different data depending on the values passed to it. This makes components flexible and reduces code duplication.

I also learned how to access props within functional components and display their values using JSX. By passing different data to the same component, I saw how React can reuse a single component in multiple places while producing different outputs.

To practice these concepts, I created reusable components that displayed different information based on the props they received. This exercise helped me understand the flow of data from parent to child components and the importance of designing components that can be reused throughout an application.

### What I Learned

* Understanding React props
* Passing data from parent to child components
* Accessing props in functional components
* Rendering dynamic content using props
* Creating reusable React components
* Improving code organization through component reuse

Today's session helped me understand how data flows within a React application. Learning about props made it clear how React encourages reusable and maintainable components, laying the groundwork for building larger and more dynamic user interfaces.
