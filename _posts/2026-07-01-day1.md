Today was the first day of my TypeScript learning journey. I started following the Code with Mosh TypeScript course to build a stronger foundation in modern JavaScript development. The first lesson was mainly about understanding what TypeScript is, why it was introduced, and how it helps developers write more reliable and maintainable code.

I learned that TypeScript is a superset of JavaScript, which means every valid JavaScript program is also valid TypeScript. The biggest advantage of TypeScript is static type checking, which helps catch many errors during development instead of finding them later while running the application. This makes debugging easier and improves the overall development experience, especially in large projects.

After understanding the theory, I set up my development environment. I installed the TypeScript compiler using npm, verified the installation, and created my first TypeScript file. I also learned that browsers and Node.js cannot execute TypeScript directly. Instead, the TypeScript compiler (`tsc`) converts `.ts` files into standard JavaScript files, which can then be executed normally.

### Commands Used

```bash
npm install -g typescript
tsc --version
tsc index.ts
node index.js
```

### What I Learned

- What TypeScript is and why it was created
- TypeScript is a superset of JavaScript
- Benefits of static type checking
- Difference between TypeScript and JavaScript
- How TypeScript is compiled into JavaScript
- Installing and using the TypeScript compiler (`tsc`)
- Creating and running the first TypeScript program

Although today's session was mostly introductory, it gave me a clear understanding of why TypeScript has become the preferred choice for many modern web development projects. I'm looking forward to learning about data types, variables, and type annotations in the next lesson.
