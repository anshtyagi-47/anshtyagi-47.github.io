Today I learned about **props**, one of the most important concepts in React. Props (short for properties) allow data to be passed from a parent component to a child component, making components reusable and dynamic.

Instead of hardcoding values inside components, I learned how to pass different data through props and access it inside child components. This makes it possible to reuse the same component multiple times with different content.

I also understood that props are **read-only**, which means a child component cannot modify the data it receives. This one-way flow of data helps keep React applications predictable and easier to debug.

To practice, I created a few simple components and passed different values such as names, titles, and numbers through props. It helped me understand how React applications are built by combining multiple reusable components.

### What I Learned

- What props are in React
- Passing data from parent to child components
- Accessing props inside components
- Reusing components with different data
- One-way data flow in React
- Why props are read-only

Today's lesson showed me how React makes UI development modular and reusable. Understanding props feels like an important step before moving on to state management and creating more interactive applications.
