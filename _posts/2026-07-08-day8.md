Today I learned about **state** in React and how to manage dynamic data using the **`useState`** hook. Unlike props, which are used to pass data between components, state allows a component to store and update its own data, making the user interface interactive.

I started by understanding why state is important in React applications. Whenever the state of a component changes, React automatically re-renders the component to display the updated information. This makes it easy to create dynamic applications that respond to user interactions.

I explored the syntax of the **`useState`** hook and learned how to initialize state variables, update their values, and use them within JSX. I also understood that state should always be updated using the provided setter function instead of modifying it directly.

To practice these concepts, I built a simple **counter application** where users could increase and decrease a value. I also created basic form examples that updated state as the user entered information. These exercises demonstrated how state changes immediately reflect in the user interface.

### What I Learned

* Understanding state in React
* Introduction to the `useState` hook
* Creating and updating state variables
* React re-rendering on state changes
* Building a simple counter application
* Managing form input using state
* Writing interactive React components

Today's session introduced one of the most essential concepts in React development. Learning how to use the `useState` hook helped me understand how React applications become interactive and respond efficiently to user actions. This knowledge will be fundamental as I continue building more complex applications.
