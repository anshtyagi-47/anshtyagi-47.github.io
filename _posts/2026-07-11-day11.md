Today I learned how React handles forms and user input. I explored controlled components, where the value of an input field is managed using React state instead of the DOM. This approach makes it easier to validate input, update the UI, and keep the application's data in sync.

I practiced handling different input fields using the `onChange` event and updating state with the `useState` hook. I also learned how to handle form submission using the `onSubmit` event and prevent the default page refresh with `preventDefault()`.

Building a simple form helped me understand how user input flows through a React application and how state plays a key role in creating interactive forms.

### What I Learned

- Controlled components
- Managing form data with `useState`
- Handling `onChange` events
- Form submission with `onSubmit`
- Using `preventDefault()`
- Keeping form inputs synchronized with state

Today's lesson showed me how React simplifies working with forms while giving developers full control over user input. These concepts will be useful when building real-world applications that require login forms, registration pages, or user data collection.
