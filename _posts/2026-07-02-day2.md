Today I continued learning TypeScript by exploring its basic data types and some of the core language features. I learned how TypeScript allows us to define the type of variables, making the code more predictable and reducing the chances of runtime errors.

The session covered primitive types like `number`, `string`, and `boolean`, along with arrays, tuples, enums, and the `any` type. I also learned about type inference, where TypeScript automatically determines the type of a variable based on its assigned value. This helps keep the code clean while still providing type safety.

One interesting concept was tuples, which allow storing multiple values of different types in a fixed order. I also understood why the `any` type should be used carefully, as it disables TypeScript's type checking for that variable.

### What I Learned

- Primitive data types (`number`, `string`, `boolean`)
- Arrays and tuples
- Enums
- The `any` type
- Type inference
- Variable type annotations

Today's lesson helped me understand how TypeScript improves code quality by making data types explicit. I'm becoming more comfortable with the syntax and looking forward to learning about functions, objects, and interfaces in the next lesson.
