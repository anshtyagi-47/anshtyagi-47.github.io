On the third day of my TypeScript training, I learned how TypeScript handles functions and objects. The lesson focused on writing cleaner and more reliable code by specifying the types of function parameters and return values.

I learned that adding type annotations to functions helps prevent common mistakes, such as passing the wrong type of argument or returning an unexpected value. I also explored how to define the structure of objects by specifying the type of each property, making the code easier to understand and maintain.

Another useful concept was optional parameters and default parameter values, which make functions more flexible while still keeping them type-safe. Throughout the session, I practiced writing small examples to get comfortable with the syntax.

### What I Learned

- Function parameter type annotations
- Function return types
- Optional and default parameters
- Defining object types
- Type checking while working with objects
- Writing cleaner and more maintainable TypeScript code

Today's lesson showed how TypeScript can catch errors during development instead of at runtime. Understanding functions and object types feels like an important step toward building larger applications with confidence.
