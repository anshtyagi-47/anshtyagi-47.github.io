Today I learned about two of the most important concepts in React: **components** and **JSX**. These form the foundation of every React application and make it possible to build user interfaces in a modular and reusable way.

I started by understanding what React components are and why they are considered the building blocks of a React application. Instead of writing an entire webpage in one file, React allows us to divide the interface into smaller, reusable components. This approach makes the code easier to organize, maintain, and reuse across different parts of an application.

Next, I explored **JSX (JavaScript XML)**, which allows HTML-like syntax to be written directly inside JavaScript. I learned how JSX makes UI code more readable while still giving access to JavaScript expressions using curly braces (`{}`). I also understood that JSX is converted into regular JavaScript before being rendered in the browser.

To reinforce these concepts, I created a simple static user interface using multiple React components. This helped me understand how components can be combined to build a complete webpage while keeping the code clean and modular.

### What I Learned

* Introduction to React components
* Functional components in React
* Understanding JSX syntax
* Embedding JavaScript expressions in JSX
* Returning JSX from components
* Creating and organizing reusable components
* Building a simple static UI using React

Today's session gave me a solid understanding of how React applications are structured. Learning about components and JSX showed me how React simplifies UI development by making code reusable, organized, and easier to maintain. These concepts will serve as the foundation for more advanced React topics in the coming days.
