Today I learned about **state**, one of the most important concepts in React. Unlike props, which receive data from a parent component, state allows a component to manage and update its own data. I was introduced to the `useState` hook and learned how it helps React re-render the UI whenever the state changes.

I also explored event handling in React. I learned how to respond to user actions such as button clicks and input changes by attaching event handlers to JSX elements. Combining state with events makes it possible to create interactive applications where the user interface updates instantly based on user interactions.

To reinforce these concepts, I practiced building simple examples like a counter and handling input values. These exercises helped me understand how React keeps the UI synchronized with the underlying data.

### What I Learned

- What state is in React
- Using the `useState` hook
- Updating state correctly
- How React re-renders components
- Handling events like button clicks and input changes
- Building interactive components with state

Today's lesson made React feel much more dynamic. I can now create components that respond to user actions instead of displaying only static content, which is a big step toward building real-world applications.
